Службы:
- ins-product-aggregator (ipa)
- ins-comp-settlement (ics)
опрашивает страховые компании (СК) только из k8sc определенного одного региона, чтобы:
- в БД не поступали повторные данные
- не выполнять одинаковую работу на всех кластерах
- не реплицировать избыточные данные
- не загружать api СК
Прочие регионы получают аггрегаты из СК при репликации.
У службы core-app предусмотреть версию, в которой запрос каталога продуктов и реестра страховок происходит напрямую из ics-bd. Данную версию core-app деплоить на всех регионах кроме региона, делающего запросы в СК.
